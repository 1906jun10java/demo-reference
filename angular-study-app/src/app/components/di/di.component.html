Instantiating a class in another class strictly ties one with the other.
To remove this issue, we use Dependency Injection. This is a coding pattern in which a class receives the instances of the objects it needs (called dependencies) from an external source rather than creating them itself
Angular can give away instances with all its dependencies fixed.
Normally, we inject Services into Components.
{{subTitleInjectable}}
We can make a class available for DI with the @Injectable decorator.
We can use the Service as parameters in our Component.
Angular will inject the service when the constructor gets called.
